Chapter 5 - Events

This chapter covers various events that can make your app more interactive. You will learn how to use timers and other time-related tools to make your app more interactive. You will also be introduced to the concept of object-oriented programming, which is a powerful way to organize your code.

Time​

In programming, time is a way to measure how long it takes for things to happen. It's like a clock that keeps track of how much time has passed since a certain event.

In app development, time is important because many apps need to perform actions at certain times or intervals. For example, a weather app might need to update the forecast every hour, or a game might need to check if the player has completed a task after a certain amount of time has passed.

Developers use timers to make sure that events happen at the right time. A timer is like a stopwatch that can be set to go off after a certain amount of time has passed.

Using time and timers properly in app development helps make sure that everything happens at the right time and in the right order. This makes apps more reliable and useful for users.

React Animated Timing​

React's Animated library is a tool for creating animations and other types of interactions with React components. It is a powerful tool that can be used to create complex animations and interactions.

Animated.timing allows you to change the properties of a component over time, creating smooth and engaging animations. Animated.timing works by specifying the starting and ending values for a property of a component, and the duration over which the animation should take place. The tool then automatically calculates and updates the values of that property over time, resulting in a smooth transition between the starting and ending values.

For example, if you wanted to create an animation that moves a component from one position on the screen to another, you could use Animated.timing to change the component's position property over a specified duration. As the animation plays, the component's position would gradually change, resulting in a smooth, visually appealing effect.

In the example below, we use animated.timing to create an animation that fades a component in and out over a duration of 1 second. The component's opacity property is set to 0 at the start of the animation, and 1 at the end. As the animation plays, the component's opacity gradually changes from 0 to 1, resulting in a smooth fade-in effect.

You might notice that we create a new kind of View, called FadeInView. This is a custom component that we created to make it easier to create animations with Animated.timing. Let's take a closer look at this line, const FadeInView: React.FC<FadeInViewProps> = props =>.

The FadeInView: React.FC<FadeInViewProps> part is called a type declaration. In this case, const FadeInView declares a new constant variable named FadeInView, which is assigned the value of the function definition that follows it. It specifies that the FadeInView component is a function component that receives props of type FadeInViewProps. React.FC stands for "React function component".

The props => part is the function's parameter list. It specifies that the component function accepts a single parameter called props.

Gesture​

A gesture is a movement or action that a user makes with their hands or fingers. For example, a user might tap the screen to select an item, or swipe their finger across the screen to scroll through a list. In the simulator, you can simulate a gesture by dragging your mouse or trackpad across the screen.

PanResponder is a React Native API that provides a way to handle gestures such as touches, swipes, and pinches. It allows the developer to respond to gestures in a flexible way, by providing callbacks for each step of the gesture.

When a component is given a PanResponder, it is able to track the user's gestures and provide information about their movement, velocity, and location. This allows the developer to implement custom logic for different gestures, such as dragging an element across the screen or pinching to zoom in on an image.

In the example below, we use PanResponder to create a component that can be dragged across the screen. The component's position is updated as the user drags their finger across the screen, and it snaps back to its original position when the user releases their finger.

The Animated.View component has a transform property that is set to an array of two objects: {translateX: pan.x} and {translateY: pan.y}. These objects are used to animate the position of the blue box when it is dragged.

The pan variable is created using the useRef hook, and it is set to a new Animated.ValueXY() object. This object represents the position of the blue box on the screen.

The panResponder variable is also created using the useRef hook, and it is set to the result of calling PanResponder.create(). This creates a new PanResponder object that responds to touch gestures on the screen. We will discuss the useRef hook in more detail in a bit.

The onMoveShouldSetPanResponder method of the panResponder object is set to a function that always returns true. This means that the panResponder object will always respond to touch gestures.

The onPanResponderMove method of the panResponder object is set to an Animated.event function. This function takes an array of two arguments: null and an object with the properties dx and dy. The dx and dy properties represent the change in position of the blue box when it is dragged. This function updates the pan variable to reflect the new position of the blue box.

The onPanResponderRelease method of the panResponder object is set to a function that animates the position of the blue box back to its original position. This function uses the Animated.spring function to create a spring animation that moves the blue box back to {x: 0, y: 0}.

useRef hook​

useRef is a special tool in React that helps us access a piece of data in our components. We can use it to hold onto some data and access it later when we need it, just like a box where we can put something in and take it out later.

In the example code, const pan = useRef(new Animated.ValueXY()).current; creates a box called pan that holds the animated value of the X and Y coordinates of our box.

We can also use useRef to keep track of something that needs to be persisted between renderings, like an animation object or an HTML element. useRef is similar to useState which we have used before. The difference is that useRef does not trigger a re-render when its content changes.

Objective-oriented programming​

In the class component example, the pan and panResponder variables are created as properties of the App class. This is called object-oriented programming. In object-oriented programming, we create objects that have properties and methods. The properties and methods of an object are called members.

Object-oriented programming (OOP) is a programming paradigm that revolves around the concept of "objects" that have properties and behaviors. It's a way to organize and structure code in a more intuitive and manageable way, allowing for better code reuse, abstraction, and encapsulation.

In OOP, everything is an object, and objects are instances of classes. A class is like a blueprint or a template that defines the properties and behaviors of objects. For example, if you have a class called "Car", you can create many instances of that class, each with its own properties like make, model, and color, and behaviors like accelerating, braking, and turning.

The main advantages of OOP are:

  1. Code reuse: OOP allows us to reuse code by creating objects that share the same properties and behaviors. For example, if we have a class called "Car", we can create many instances of that class, each with its own properties like make, model, and color, and behaviors like accelerating, braking, and turning.
  2. Abstraction: OOP allows us to hide the implementation details of an object from the outside world. This allows us to focus on the important parts of the object, and not worry about the details. For example, a car driver doesn't need to know how the engine works, but only needs to know how to start and stop the car.
  3. Encapsulation: OOP allows us to restrict access to certain properties and methods of an object. This allows us to hide the implementation details of an object from the outside world, and only expose the properties and methods that we want to be accessible. We can use encapsulation to hide the internal details of the car and only provide access to its behavior through its methods. For example:
class Car {
  #color = 'red'; // private property
  #engineOn = false; // private property

  startEngine() {
    this.#engineOn = true;
    console.log('Engine started');
  }

  stopEngine() {
    this.#engineOn = false;
    console.log('Engine stopped');
  }

  get color() {
    // public getter
    return this.#color;
  }

  set color(value) {
    // public setter
    this.#color = value;
    console.log(`Color changed to ${this.#color}`);
  }
}

const myCar = new Car();

myCar.color = 'blue'; // calling the public setter
console.log(myCar.color); // calling the public getter
myCar.startEngine(); // calling the public method

In this example, we have used the # symbol to mark the color and engineOn properties as private, which means they can only be accessed from within the class. We have also provided public access to the color property using a getter and setter method, which allows us to modify and retrieve the color of the car while hiding the implementation details.

  1. Inheritance: Inheritance is the process of creating new classes based on existing classes, which allows us to reuse existing code and build new classes that inherit the properties and methods of their parent class.

Let's say we want to create a new class called SportsCar that inherits from our Car class and adds some new properties and methods that define its unique behavior:

class SportsCar extends Car {
  #topSpeed = 200; // private property

  accelerate() {
    console.log(`Accelerating to ${this.#topSpeed} km/h`);
  }
}

const mySportsCar = new SportsCar();

mySportsCar.color = 'green';
console.log(mySportsCar.color); // calling the public getter
mySportsCar.startEngine(); // calling the public method
mySportsCar.accelerate(); // calling the public method of the derived class

In this example, we have created a new SportsCar class that inherits from the Car class using the extends keyword. We have also added a private topSpeed property and a public accelerate() method that define the unique behavior of a sports car.

Class Properties​

A class component has a render() method that returns a React element, which can be rendered to the screen. It also has access to a state object, which is used to manage the component's data and re-render the component when the data changes.

Class components can also have lifecycle methods, which are special methods that are called at specific points during the component's lifecycle, such as when it's first created or when it's updated. These methods can be used to perform tasks such as fetching data or updating the component's state.

Here's an example of a simple class component in React:

import React, {Component} from 'react';

class Greeting extends Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

export default Greeting;

In this example, we're creating a Greeting component that extends the Component class provided by React. It has a render() method that returns a h1 element containing a greeting message with the name prop passed in as a parameter. We're also exporting the Greeting component so that it can be used in other parts of the application.
