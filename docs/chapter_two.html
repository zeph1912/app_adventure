Chapter 2 - Components

This chapter will introduce you to different React components that you can use to build your app. The components that we will cover in this chapter include: Text, TextInput, Image, and Button. We will learn how to use these components to create a simple app that displays a text field, an image, and a button.

What is a React component?​

A React component is a piece of code that you can use to build your app. React components are like building blocks that you can use to build your app. You can use React components to create buttons, text fields, images, and more. Each component is responsible for rendering a specific part of your user interface, and they can be combined to create more complex UI elements.

For example, you might create a component to display a button, and another component to display a form. You could then combine these components to create a login screen. This allows you to reuse the same button and form components throughout your app, which can save you a lot of time and make your code easier to maintain.

State​

In React, components are like characters in a story. Properties are like instructions you give the characters to help them act out their role. State is like a character's thoughts or feelings that can change over time.

For example, imagine two hungry cats in a cat cafe. We can use state to keep track of their hunger status. When we press their "feed" button, it updates their hunger state.

To add state to a component, we use a special function called "useState". It's like a magic spell that lets us give our components a memory.

Here is how we can use the useState function to add state to the ShowButton component.

Let's take a closer look on useState function. The useState function takes a parameter, which is the initial value of the state. In the code example above, the initial value of the state is true. The useState function returns an array with two elements. The first element is the current value of the state, and the second element is a function that can be used to update the state.

const [isHungry, setIsHungry] = useState(true);

As we said before, useState a magic spell that does two things:

  • It creates a state variable called isHungry, and sets its initial value to true.
  • It creates a function called setIsHungry, which can be used to update the isHungry state. You don't need define isHungry and setIsHungry yourself.

For example, whenever you call setIsHungry(false), the isHungry state will be updated to false.

The component then uses the isHungry state to display a message that says whether or not the cat is hungry:

<Text>I am a cat, and I am {isHungry ? 'hungry' : 'full'}!</Text>

The component also creates a button that can be pressed to feed the cat. The button is disabled if the cat is not hungry:

<Button
  onPress={() => {
    setIsHungry(false);
  }}
  disabled={!isHungry}
  title={isHungry ? 'Pour me some milk, please!' : 'Thank you!'}
/>

When the button is pressed, it calls the setIsHungry function to update the isHungry state to false, which makes the cat full. The button is then disabled and its text changes to "Thank you!" to indicate that the cat has been fed.

Why properties?​

Properties making components reusable. For example, we can create a Cat component that takes a name property, and then use it to create multiple cats with different names. Otherwise, we would have to create a separate component for each cat, which doubles the amount of work.

Core Components​

React Native comes with a few components ready to use. These components are called "Core Components". Here is a list of some of the most commonly used Core Components:

React Native UI ComponentAndroid ViewiOS ViewWeb AnalogDescription
<View><ViewGroup><UIView>A non-scrolling <div>A container that supports layout with flexbox, style, some touch handling, and accessibility controls
<Text><TextView><UITextView><p>Displays, styles, and nests strings of text and even handles touch events
<Image><ImageView><UIImageView><img>Displays different types of images
<ScrollView><ScrollView><UIScrollView><div>A generic scrolling container that can contain multiple components and views
<TextInput><EditText><UITextField><input type="text">Allows the user to enter text

In the Android View, iOS View, and Web Analog columns, we have listed the corresponding native views for each React Native component.

Native views means the views that are defined by the platform itself, and can only be used on that platform. The good thing is that React Native provides a unified interface for all the native views, so you don't need to learn the platform-specific APIs.

To use any view provided by React, we need to import it from the react-native package. For example, to use the Text component, we write:

import {Text} from 'react-native';

Similarly, to use the core React features and the useState magic, we write:

import React, {useState} from 'react';

The Text component displays the translated text. The split() method is used to split the text into an array of words. The map() method is then called on the array to transform each word into a pizza emoji. If the word is an empty string, it returns false and is ignored. Otherwise, it returns a pizza emoji. The join() method is used to join the array back into a string with spaces between each emoji.
