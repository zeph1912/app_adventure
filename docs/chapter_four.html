Chapter 4 - Layout

This chapter will cover the basics of layout in React. You will use the Flexbox layout system to create a responsive layout for your app. Specifically, you will learn how to use the View component to create a container for other components, and different properties to control the size and position of the container and the components inside it.

Layout​

In a React app, layout refers to the way different components are arranged on the screen. Components are the building blocks of an app, and each component can be considered as a small part of the app that performs a specific task.

As we have seen in the previous chapter, we can use the View component to create a container for other components. We can then use the Text and other components to display contents on the screen. We have also learned to use size, flex and percentage values to control the size of the container and the components inside it.

In this chapter, we will learn more about layout in React. We will use the Flexbox layout system to create a responsive layout for our app.

Flexbox direction​

First, let's review flex. In React, you can use flex to control how much space your items take up in a container. The more flex you give an item, the more space it will take up compared to other items in the container.

In the example below, we have a container with three items. If you give the container a flex value of 1, you're telling it to share the available space equally among the boxes. But if you give the red box a flex value of 1, the orange box a flex value of 2, and the green box a flex value of 3, you're telling the container to divide the space in a ratio of 1:2:3.

So if there are 6 units of space available, the red box will take up 1 unit, the orange box will take up 2 units, and the green box will take up 3 units.

In this example, we introduce a new property called flexDirection. The flexDirection property is used to decide how the components are laid out in relation to each other. The main axis is the direction in which the components are laid out, while the cross axis is the direction perpendicular to that.

By default, flexDirection is set to column. This means that the components are laid out from top to bottom. If we set flexDirection to row, the components will be laid out from left to right. flexDirection can also be set to column-reverse, which means the components will be laid out from bottom to top. Finally, flexDirection can be set to row-reverse, which means the components will be laid out from right to left.

TouchableOpacity​

Button and TouchableOpacity are two different ways to create clickable elements in a mobile app. The main difference between them is that Button is a pre-made component that comes with default styles and behavior, while TouchableOpacity is a more flexible component that can be customized to suit the app's design.

Button is a simple component that is usually used for basic actions like submitting a form or navigating to a new screen. It has a standard look and feel, with a rectangular shape, text label, and background color that is determined by the platform (e.g., blue on iOS, gray on Android). Button also has built-in accessibility features like the ability to be focused and activated using the keyboard.

TouchableOpacity, on the other hand, is a customizable component that can be styled to look and behave in a variety of ways. It has a transparent background by default, which allows the underlying content to show through. TouchableOpacity can be styled with custom colors, borders, and animations to create a unique look and feel for the app. It can also be used to trigger more complex actions like displaying a dropdown menu or opening a modal window.

In summary, Button is a simpler, pre-made component that is good for basic actions, while TouchableOpacity is a more flexible and customizable component that can be used for more complex actions and customized to match the app's design.

Flexbox Justify Content​

The justify content property in Flexbox helps control how items are arranged horizontally within a container. It can help you decide how much space to put between items, or how to center items within the container. For example, you can tell the items to be centered in the middle of the container, or to be evenly spaced out from each other. This makes it easy to create a nice-looking layout for your app.

The main axis is the axis along which the Flexbox container is laid out, and the direction of the main axis is determined by the flex-direction property. For example, if flex-direction is set to row, then the main axis will be horizontal, and if it's set to column, the main axis will be vertical.

Justify-content can take several values, each of which determines how the items within the container are aligned along the main axis. Here are the possible values:

  • flex-start: Items are aligned at the beginning of the container. This is the default value.
  • flex-end: Items are aligned at the end of the container.
  • center: Items are centered within the container.
  • space-between: Items are evenly distributed within the container, with equal space between them.
  • space-around: Items are evenly distributed within the container, with equal space around them.
  • space-evenly: Items are distributed so that there is equal space between them and around them.

For example, if justify-content is set to center and flex-direction is set to row, then the items in the container will be centered horizontally. If flex-direction is set to column instead, then the items will be centered vertically.

Dynamic Flexbox sizes​

In app development, dynamic resizing of a component means that the component can adjust its size automatically based on the available space in the app. This is really important because it allows the app to work on many different devices with different screen sizes. For example, a phone has a smaller screen than a tablet, so a component that looks good on a phone might be too small on a tablet. If the component is set up to resize dynamically, it can expand to fill more space on the larger screen, so that it still looks good and is easy to use.

Dynamic resizing can also help make the app more flexible and responsive to user input. For example, if a user rotates their device from landscape to portrait mode, the screen size changes, and if the components are set up to resize dynamically, they will adjust automatically to fit the new screen size. This can make the app feel more user-friendly and intuitive.

Also, dynamic resizing can help make the app more accessible. For example, if a user has a vision impairment, they might want to increase the size of the text in the app. If the components are set up to resize dynamically, they will adjust automatically to fit the new text size, so that the app is still easy to use.

Dynamic Flexbox component sizes​

Now, let's look at how to resize the components dimensions (width and height) dynamically. The dimensions of a component can be controlled using the width and height properties. The width property controls the width of the component, and the height property controls the height of the component.

The example below uses two useState hooks to store the values of the width and height properties. We have three ways to set the value of these properties:

  • auto - The component will be sized to fit its content.
  • pixel value - The component will be sized to the specified number of pixels.
  • percentage value - The component will be sized to the specified percentage of the parent component.

If use auto as the value of the width or height properties, the component will be sized to fit its content. For example, if the component contains a text element, the component will be sized to fit the text. If the component contains a button element, the component will be sized to fit the button.
